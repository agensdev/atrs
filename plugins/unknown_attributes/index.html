Unknown Attributes

unknown_attributes will by default raise an error if a class is initialized with unhandled attributes. An optional callback lambda can be applied to override the default behavior.

Example

class Foo
  include Wardrobe
  plugin :unknown_attributes
  attribute :bar, String
end

Foo.new(bar: 'Test')
#=> #<Foo:0x007fc34a142160 @bar="Test">

Foo.new(bar: 'Test', missing: 'Lorem ipsum', random: 'Value')
#=> Wardrobe::Plugins::UnknownAttributes::UnknownAttributesError: missing, random

Custom callback lambda

class Bar
  include Wardrobe
  plugin :unknown_attributes, callback: ->(attributes, instance) { instance.missing = attributes }
  attribute :foo, String
  attribute :missing, Hash
end

Bar.new(foo: 'Test', a: 1, b: 2, c: 3)
#=> #<Bar:0x007fc0fb8d8998 @foo="Test", @missing={:a=>1, :b=>2, :c=>3}>
