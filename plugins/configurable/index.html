Configurable

Configurable adds powerful class level configuration to your wardrobe models.

Example

Start by creating a normal Wardrobe model with the immutable plugin added.

class Config
  include Wardrobe
  plugin :immutable
  plugin :presenter

  attribute :foo, String
  attribute :children, Array[Config]
end

Then create another Wardrobe model and enable the configurable plugin. Enable the Config class by calling #configurable method. It takes 3 arguments. The first is the name of the method that will enable to you to access the config. The second is the name of the method used to mutate the config. The third argument is the config class created above. You can add multiple “configurables”. Just give them different method names.

class User
  include Wardrobe
  plugin :configurable
  configurable :config, :configure, Config
end

Now you can configure your class by calling User.configure with a block.

User.configure do |c|
  c.foo = 'bar'
  c.children << { foo: 'bar child'}
end

You can safely inherit from the User class. Any configurable will be inherited but will never be able to change to config in the parent.

class InheritedUser < User; end

InheritedUser.configure do |c|
  c.foo = 'bar in InheritedUser'
  c.children[0].foo = 'First child changed in InheritedUser'
  c.children << { foo: 'Added in InheritedUser'}
end

User.config._present #=> {:foo=>"bar", :children=>[{:foo=>"bar child", :children=>[]}]}
InheritedUser.config._present #=> {:foo=>"bar in InheritedUser", :children=>[{:foo=>"First child changed in InheritedUser", :children=>[]}, {:foo=>"Added in InheritedUser", :children=>[]}, {:foo=>"Added in InheritedUser", :children=>[]}]}
