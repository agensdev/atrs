Coercion

Coercion is the only plugin in Wardrobe enabled by default. Coercion works with most of Rubys types. Coercion will also happen when mutating Array, Hash and Set.

Example

class User
  include Wardrobe
  attribute :id, Integer
  attribute :name, String
  attribute :status, Symbol
  attribute :friends, Array[User]
  attribute :interests, Hash[String => Symbol]
end

user = User.new(
  id: 1.1,
  name: :'Example User',
  status: 'active',
  friends: [
    {
      id: '0045',
      name: 'Another User',
      status: 'inactive'
    }
  ],
  interests: {
    'architecture' => 'medium',
    :sports => 'low',
    :travel => :high
  }
)

# <User:0x007fcc160851f8
#   @id=1,
#   @name="Example User",
#   @status=:active,
#   @friends=[
#     <User:0x007fcc16084b68
#       @id=45,
#       @name="Another User",
#       @status=:inactive,
#       @friends=[],
#       @interests={}>
#   ],
#   @interests={
#     "architecture"=>:medium,
#     "sports"=>:low,
#     "travel"=>:high
# }>

Mutating Array

Given the example above adding a friend to the friends array will coerce the given hash to a User object. <<, push, unshift and insert will all work with coercion.

user.friends << { id: '22', name: 'Added later' }
# => [
#  #<User:0x007fb242c0c960
#    @id=45,
#    @name="Another User",
#    @status=:inactive,
#    @friends=[],
#    @interests={}>,
#  #<User:0x007fb242bd66a8
#    @id=22,
#    @name="Added later",
#    @status=nil,
#    @friends=[],
#    @interests={}>
# ]

Mutating Hash

Mutating a hash with coercion is supported with the [], merge! amd store Hash methods.

Mutating Set

Mutating a set with coercion is supported with the <<, add amd merge Set methods.

Disabling coercion

Disable coercion for a class

class Foo
  include Wardrobe(coerce: false)

  attribute :bar, String
end

Foo.new(bar: :a_symbol).bar # => :a_symbol

Disable coercion for a single attribute

class Foo
  include Wardrobe

  attribute :bar, String, coerce: false
end

Foo.new(bar: :a_symbol).bar # => :a_symbol
