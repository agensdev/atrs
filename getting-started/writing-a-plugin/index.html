Writing a plugin

Lets create a plugin that capitalizes strings.Wardrobe uses a middleware inspired system with blocks of code that manipulate data when “setting” or “getting” the value for an attribute. You can create your own setters and getters. We’ll start by creating a setter that will apply to all attributes with the capitalize: true option.

require 'wardrobe'

Wardrobe.register_setter(
 name: :capitalize,
 before: [:setter],
 after: [:coercer],
 use_if: ->(atr) { atr.options[:capitalize] && atr.klass == String },
 setter: lambda do |value, _atr, _instance|
   value ? value.capitalize : value
 end
)

The before and after arrays allow you to tell Wardrobe about other setters that need to happen before or after the one you are creating.

The use_if lambda ensures that the setter middleware is only added to attributes that meet a given criteria.

The setter is a lambda that takes three arguments. The value passing through the middleware, the attribute instance and the Wardrobe model instance.

Now lets write the the rest of the plugin:

module Capitalize
 extend Wardrobe::Plugin
 option :capitalize, Wardrobe::Boolean, setter: :capitalize
end

Wardrobe.register_plugin(:capitalize, Capitalize)

Usage

class Person
 include Wardrobe
 plugin :capitalize

 attribute :name, String, capitalize: true
 attribute :gender, String
end

Person.new(name: 'foo', gender: 'male')
#=> #<Person:0x007fba42a273b8 @name="Foo", @gender="male">
